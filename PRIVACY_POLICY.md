# 🔒 Dingle 개인정보 처리방침

**최종 업데이트**: 2026년 2월 1일  
**버전**: v1.3.0

---

## 📌 요약

**Dingle은 사용자의 개인정보를 수집하지 않습니다.**

- ✅ 모든 데이터는 사용자 컴퓨터에만 저장됩니다
- ✅ 외부 서버로 데이터를 전송하지 않습니다
- ✅ 사용자를 추적하지 않습니다
- ✅ 분석 도구를 사용하지 않습니다

---

## 1. 수집하는 정보

### ❌ Dingle이 수집하지 않는 정보

- 이름, 이메일, 전화번호
- 주소, 위치 정보
- 결제 정보
- 사용 패턴, 행동 데이터
- 다이어리 내용
- 사진, 스티커, 텍스트
- 기기 정보 (OS, 브라우저 등)

### ✅ Dingle이 저장하는 정보 (로컬만)

다음 정보는 **사용자 컴퓨터에만** 저장되며, 외부로 전송되지 않습니다:

- 다이어리 데이터 (텍스트, 사진, 스티커, 레이아웃)
- 사용자 설정 (테마, 폰트, 위젯)
- 백업 파일

**저장 위치**:
- Windows: `C:\Users\[사용자명]\AppData\Local\dingle`
- macOS: `~/Library/Application Support/dingle`
- Linux: `~/.config/dingle`

---

## 2. 데이터 사용 방식

### 로컬 저장만

Dingle은 **완전한 오프라인 앱**입니다:

- 모든 데이터는 사용자 컴퓨터의 파일 시스템에 JSON 형식으로 저장됩니다
- 클라우드 서버, 데이터베이스, 외부 스토리지를 사용하지 않습니다
- 다른 사용자와 데이터를 공유하지 않습니다

### 데이터 접근 권한

- **사용자만** 자신의 데이터에 접근할 수 있습니다
- Dingle 개발팀은 사용자 데이터에 접근할 수 없습니다
- 제3자에게 데이터를 제공하지 않습니다

---

## 3. 네트워크 통신

Dingle은 다음 목적으로만 인터넷을 사용합니다:

### ✅ 필수 기능

| 기능 | 도메인 | 목적 | 데이터 전송 |
|------|--------|------|------------|
| 오하아사 운세 | `asahi.co.jp` | 일일 운세 데이터 가져오기 | 별자리 정보만 |
| 자동 업데이트 | `github.com` | 최신 버전 확인 및 다운로드 | 현재 버전 번호만 |

### ✅ 선택 기능 (사용자가 추가한 경우만)

| 기능 | 도메인 | 목적 | 데이터 전송 |
|------|--------|------|------------|
| 트위터 임베드 | `twitter.com` | 트윗 표시 | URL만 (공식 API) |
| 인스타그램 임베드 | `instagram.com` | 포스트 표시 | URL만 (공식 API) |
| YouTube 임베드 | `youtube.com` | 동영상 재생 | 동영상 ID만 |
| Spotify 임베드 | `spotify.com` | 음악 재생 | 트랙 ID만 |
| SoundCloud 임베드 | `soundcloud.com` | 음악 재생 | 트랙 URL만 |
| Pinterest 임베드 | `pinterest.com` | 핀 표시 | 핀 URL만 |

**중요**: 
- 임베드 기능은 사용자가 직접 URL을 추가한 경우에만 작동합니다
- 다이어리 내용은 전송되지 않습니다
- 각 플랫폼의 공식 API만 사용합니다

## 4. 쿠키 및 추적

### ❌ 사용하지 않는 것들

- Google Analytics
- Facebook Pixel
- 광고 추적 코드
- 사용자 행동 분석 도구
- 세션 추적
- 쿠키 (임베드 플랫폼 제외)

### ⚠️ 제3자 쿠키

임베드 기능 (트위터, 인스타그램 등)을 사용하는 경우, 해당 플랫폼의 쿠키가 설정될 수 있습니다. 이는 각 플랫폼의 개인정보 처리방침을 따릅니다:

- [Twitter 개인정보 처리방침](https://twitter.com/privacy)
- [Instagram 개인정보 처리방침](https://help.instagram.com/519522125107875)
- [YouTube 개인정보 처리방침](https://policies.google.com/privacy)

## 5. 데이터 보안

### 로컬 저장 보안

- **파일 시스템 권한**: OS의 사용자 권한으로 보호됨
- **Atomic Write**: 파일 손상 방지
- **백업 기능**: 사용자가 직접 백업 생성 및 복원

### 네트워크 보안

- **HTTPS만 사용**: 모든 네트워크 요청이 암호화됨
- **공식 API만 사용**: 신뢰할 수 있는 도메인만 접근
- **최소 권한 원칙**: 필요한 데이터만 요청

---

## 6. 데이터 보관 및 삭제

### 보관 기간

- **무제한**: 사용자가 삭제하기 전까지 로컬에 보관됩니다

### 데이터 삭제 방법

#### 방법 1: 앱 내에서 삭제
1. 설정 → 백업/복원
2. "모든 데이터 삭제" 버튼 클릭

#### 방법 2: 수동 삭제
앱을 제거한 후 다음 폴더를 삭제하세요:

- **Windows**: `C:\Users\[사용자명]\AppData\Local\dingle`
- **macOS**: `~/Library/Application Support/dingle`
- **Linux**: `~/.config/dingle`

**중요**: 폴더를 삭제하면 모든 다이어리 데이터가 영구적으로 삭제됩니다.
