## 🔒 후원자 전용 배포 전략

### 배포 방식

#### ✅ 옵션 A: 텀블벅 직접 첨부 (권장 ⭐)

**장점**:
- ✅ 후원자만 다운로드 가능
- ✅ 텀블벅 플랫폼 내에서 완결
- ✅ 접근 제어 완벽

**단점**:
- ⚠️ 자동 업데이트 안됨
- ⚠️ 파일 크기 제한 (확인 필요)
- ⚠️ 버전 업데이트 시 재업로드 필요

**실행 방법**:
```powershell
# release 폴더를 zip으로 압축
cd c:\work\digitalscrapdiary-2
Compress-Archive -Path release\Dingle*.exe,release\*.txt,release\*.md -DestinationPath Dingle_v1.3.0_Backers_Only.zip
```

---

#### ✅ 옵션 B: 구글 드라이브 + 후원자 전용 링크

**장점**:
- ✅ 파일 크기 제한 없음
- ✅ 링크 공유로 간편
- ✅ 접근 제어 가능

**단점**:
- ⚠️ 링크 유출 가능성
- ⚠️ 자동 업데이트 안됨

**실행 방법**:
1. 구글 드라이브에 폴더 생성
2. `Dingle_v1.3.0_Backers_Only.zip` 업로드
3. 링크 공유 설정: "링크가 있는 모든 사용자"
4. 링크를 텀블벅 후원자에게만 공개

---

#### ✅ 옵션 C: GitHub Private Release + 자동 업데이트

**장점**:
- ✅ 자동 업데이트 작동
- ✅ 버전 관리 용이
- ⚠️ 초기 다운로드만 제한

**단점**:
- ⚠️ 설정이 복잡
- ⚠️ 업데이트는 누구나 가능

**구조**:
```
초기 다운로드: 텀블벅 or 구글 드라이브 (후원자만)
    ↓
설치 후: GitHub Release에서 자동 업데이트 (모두 가능)
```

**장점**: 한 번 설치한 후원자는 자동으로 업데이트 받음

## 📦 배포 파일 준비

### 압축 파일 생성

```powershell
# PowerShell에서 실행
cd c:\work\digitalscrapdiary-2\release

# 필요한 파일만 선택하여 압축
$files = @(
    "Dingle Setup 1.3.0.exe",
    "Dingle 1.3.0.exe",
    "설치방법.txt",
    "README.txt",
    "업데이트_안내.txt",
    "문제해결_가이드.txt",
    "배포_안내.txt",
    "SECURITY_AUDIT_REPORT.md",
    "PRIVACY_POLICY.md",
    "SECURITY_CHECKLIST.md"
)

Compress-Archive -Path $files -DestinationPath ..\Dingle_v1.3.0_Backers_Only.zip -Force
```

### 파일 크기 확인

```powershell
Get-Item ..\Dingle_v1.3.0_Backers_Only.zip | Select-Object Name, @{Name="Size (MB)";Expression={[math]::Round($_.Length / 1MB, 2)}}
```
